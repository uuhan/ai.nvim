==============================================================================
SETUP                                                                *ai-setup*

ai.nvim uses an OpenAI-compatible chat completions endpoint.

Example:
>
  require("ai").setup({
    provider = {
      base_url = "https://api.openai.com/v1",
      api_key_env = "OPENAI_API_KEY",
      model = "gpt-4.1-mini",
      stream = true,
    },
  })
<

For local providers:
>
  require("ai").setup({
    provider = {
      base_url = "http://localhost:11434/v1",
      api_key = "",
      model = "qwen2.5-coder",
    },
  })
<

AIChat uses render-markdown.nvim for Markdown rendering. With lazy.nvim:
>
  {
    "uuhan/ai.nvim",
    dependencies = {
      {
        "MeanderingProgrammer/render-markdown.nvim",
        dependencies = { "nvim-treesitter/nvim-treesitter" },
        opts = { file_types = { "markdown" } },
      },
    },
  }
<

Install the markdown and markdown_inline Treesitter parsers for Markdown
structure. Install language parsers, such as typescript, for fenced code block
token highlighting.

==============================================================================
COMMANDS                                                          *ai-commands*

Core:
  :AI {prompt}              Ask about the visual selection or current paragraph.
  :AIExplain                Explain selected/current code.
  :AIEdit {instruction}     Generate replacement text and preview a diff.
  :AIRefactor               Refactor selected/current code.
  :AIFix                    Fix selected/current code.
  :AITest                   Suggest focused tests.
  :AIApply                  Apply the latest AI edit preview.
  :AIReject                 Clear the latest AI edit preview.

Buffer and project:
  :AIBuffer {prompt}        Ask with the current buffer as context.
  :AIFile {prompt}          Alias for :AIBuffer.
  :AISummarizeFile          Summarize current file structure.
  :AIProject {question}     Search project context with rg, then ask.
  :AIAskProject {question}  Alias for :AIProject.

Diagnostics and git:
  :AIFixDiagnostic          Ask about the diagnostic under the cursor.
  :AIFixAllDiagnostics      Ask for a patch for loaded-buffer diagnostics.
  :AIFixQuickfix            Ask about the quickfix list.
  :AIReviewDiff             Review git diff and staged diff.
  :AIExplainDiff            Explain git diff and staged diff.
  :AIFindBugInDiff          Search git diff for likely bugs.
  :AICommitMessage          Generate a commit message.

Shell commands:
  :AICmd {task}              Generate a shell command for review.
  :AIShell {task}            Alias for :AICmd.
  :AIGit {task}              Generate a git command for review.
  :AIRun                     Run the latest generated command.

Agent plan:
  :AIAgent {task}            Generate a reviewable plan.
  :AIPlanNext                Preview the next pending step.
  :AIPlanApply               Preview the next patch step.
  :AIPlanRun                 Preview the next command or test step.
  :AIPlanDone                Mark the active step done.
  :AIPlanSkip                Skip the active step.
  :AIPlanShow                Show the active plan.
  :AIPlanReset               Clear the active plan.

Chat:
  :AIChat {message}         Open side chat; optional message sends immediately.
  :AIPopChat {message}      Open floating chat; optional message sends immediately.
  :AIQuick [message]        Prompt for a short agent task and report progress
                            through notifications.
  :AIChatToggle             Open or hide side chat.
  :AIPopChatToggle          Open or hide floating chat.
  :AIChatStop               Stop the active chat or quick request.
  :AIChatReset              Clear chat history (the saved session is kept).
  :AIChatResume             Restore the most recent saved session.
  :AIChatSessions           Pick a saved session to restore.

The chat panel opens on the right. The top pane shows messages and the bottom
pane is the input area. In the input pane:
  <CR>                      Send message.
  <C-s>                     Send message.
  <S-CR> / <C-j>            Insert a line break.
  <C-l>                     Clear chat.
  <C-q>                     Close both chat panes.
  q                         Close both chat panes.
With a visual range, :'<,'>AIChat shares the selection with the conversation.
The empty input pane shows configurable ghost text from chat.placeholder.
Chat history is persisted per project as JSONL files under
stdpath("state")/ai.nvim/sessions; disable with chat.sessions.enabled = false.
Only the most recent chat.max_full_tool_results tool results are sent to the
model in full; older ones are compressed to a one-line summary.
AIChat can call harness tools by default. Tool calls and tool results are shown
as Markdown callouts in the conversation. Patch and command tools only create
previews; use :AIApply or :AIRun after inspection. Tool result details are
folded by default; use normal fold keys such as zo, zc, and za.

:AIQuick uses vim.ui.input for the prompt and the same harness tool loop as
AIChat, but reports compact status, tool-call, and short reply updates through
notifications. If fidget.nvim is available it is used directly; otherwise
ai.nvim falls back to vim.notify. Long replies open in the existing AI popup.

Harness tools:
  :AITools                  Show the model-facing Neovim tool registry.
  :AITool {name} [json]     Run one tool manually with optional JSON args.

Config:
  :AIPing                  Send a tiny request to test provider/model health.
  :AIConfig                 Show resolved ai.nvim config.
  :AIRules                  Show included project rules.

AI output buffers are reused by default. Local normal-mode keys:
  a                         Apply pending edit or patch.
  r                         Reject pending action.
  n                         Preview next agent step.
  p                         Preview next patch step.
  t                         Preview next command or test step.
  d                         Mark active plan step done.
  s                         Skip active plan step.
  q                         Close AI window.

==============================================================================
HARNESS TOOLS                                                        *ai-tools*

ai.nvim exposes a small tool registry for the coding harness. Tools are
available from Lua:
>
  local tools = require("ai").tools()
  tools.run("nvim_current_buffer", {}, function(err, result)
    print(vim.inspect(result))
  end)
<

Current tools include editor state, buffer/file reads, current selection,
diagnostics, quickfix and location lists, git diff, project file listing,
project search, structured grep/glob search, patch preview, command preview,
exact-string file edits (nvim_edit_file), and new-file previews
(nvim_create_file).

Preview tools prepare a pending action only. Patch tools still require
:AIApply, and command tools still require :AIRun. Applied edits modify
buffers; set safety.auto_write_edits = true to also write the modified
buffers to disk after a successful apply. When the preview was created by an
AIChat tool call, the :AIApply result is fed back into the conversation and
:AIReject is recorded as an editor event.

AIChat uses this registry automatically when chat.tools_enabled is true:
>
  require("ai").setup({
    chat = {
      render_markdown = true,
      tools_enabled = true,
      max_tool_rounds = 20,
      max_tool_result_chars = 20000,
      fold_tool_results = true,
    },
  })
<

==============================================================================
PROJECT RULES                                                        *ai-rules*

When present, ai.nvim includes these files in the system prompt:

  .nvim/ai.md
  .ai/rules.md
  AGENTS.md
  CLAUDE.md
  codex.md

This lets each project define stable instructions such as preferred error
handling, style constraints, and patch rules.
